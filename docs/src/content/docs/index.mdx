## Install in 30 seconds

<Tabs>
  <TabItem label="macOS / Linux">
    ```bash
    git clone https://github.com/Kristos/telemachus.git
    cd telemachus
    bun install && bun link
    export ANTHROPIC_API_KEY=sk-ant-...
    tm
    ```
  </TabItem>
  <TabItem label="Windows">
    ```powershell
    git clone https://github.com/Kristos/telemachus.git
    cd telemachus
    bun install; bun link
    $env:ANTHROPIC_API_KEY = "sk-ant-..."
    tm
    ```
  </TabItem>
  <TabItem label="Local (Ollama)">
    ```bash
    ollama pull qwen2.5-coder:7b
    git clone https://github.com/Kristos/telemachus.git
    cd telemachus && bun install && bun link
    tm
    ```
  </TabItem>
</Tabs>

## Why Telemachus?

<CardGrid>
  <Card title="Context efficient" icon="rocket">
    Orphan-safe sliding window caps the conversation at a configurable message count without ever severing `tool_use` / `tool_result` pairs.
  </Card>
  <Card title="Multi-provider" icon="puzzle">
    Anthropic (native prompt caching), OpenAI-compatible APIs, and local Ollama / LM Studio / llama.cpp. Switch mid-session with `/model`.
  </Card>
  <Card title="Cross-platform" icon="laptop">
    macOS, Linux, and **native Windows** — no WSL required. Use `TM_SHELL` to pick cmd.exe or PowerShell.
  </Card>
  <Card title="Claude Code compatible" icon="approve-check">
    Auto-mounts MCP servers from `~/.claude.json` and skills from `~/.claude/skills/`. Zero duplicate config.
  </Card>
  <Card title="Live cost tracking" icon="bars">
    Token usage, session totals, USD cost, and context-window percent right in the status bar. No surprises.
  </Card>
  <Card title="Ships as one binary" icon="setting">
    `bun build --compile` produces a single `tm` executable. No runtime required.
  </Card>
</CardGrid>
